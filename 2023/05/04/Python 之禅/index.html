Python 之禅

by Tim Peters

优美胜于丑陋。
明了胜于晦涩。
简洁胜于复杂。
复杂胜于凌乱。
扁平胜于嵌套。
宽松胜于紧凑。
可读性很重要。
即便是特例，也不可违背这些规则。
不要捕获所有错误，除非你确定需要这样做。
如果存在多种可能，不要猜测。
通常只有唯一一种是最佳的解决方案。
虽然这并不容易，因为你不是 Python 之父。
做比不做要好，但不假思索就动手还不如不做。
如果你的方案很难懂，那肯定不是一个好方案，反之亦然。
命名空间非常有用，应当多加利用。
