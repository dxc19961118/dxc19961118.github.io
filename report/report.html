使用方式

你的工作方式节奏快、迭代密，更看重迅速落地，而非事无巨细地预先规划。对 Edit 和 Bash 工具的高频使用，以及相对会话总量较低的提交次数，都表明你将 Claude Code 视作互动式的结对编程伙伴，而非批处理器。你常以简洁的指令直接切入调试与功能实现，接受首版结果可能需要打磨。这在许多会话中都有体现——例如你纠正了预订时段比较逻辑（从结束时间改为开始时间），或当 Claude 过度设计（新建数据库字段而非复用已有字段）时及时把它拉回正轨。

你的交互风格务实、直接纠偏。当 Claude 的调查陷入死胡同时，你会毫不犹豫地将其中断——无论是浏览器访问尝试，还是迟迟不出结果的深色模式调研，都是如此。这种做法有时也会带来一些摩擦，尤其是在 Hexo 复杂路由或外部 API 集成的场景下。但你总能用具体的反馈，把对话重新引回正轨。你更看重 tangible 的成果——真正跑得起来的 UI 组件、修得好的后端 Bug——而不是理论阐述；面对多轮反复的 Bug 代码，你也会一路推进，直至方案成功编译并正确运行。

核心模式：你习惯于快速的迭代式开发循环，通过直接纠偏与及时打断，引导 Claude 给出实际可用的方案，而非追求一次性完美的实现。

问题所在

主要的摩擦来自 Claude 在上下文尚未充分收集的情况下就急于生成代码，导致大量有 Bug 的迭代和在错误方向上的返工。

仓促实现
你多次允许 Claude 在未充分核实需求或现有架构的情况下，就一头扎进代码或复杂改动。一旦初始方案有误，便不得不大面积返工。若能在任何编辑开始前，先要求一份简要分析或方案概述，这类问题便可大幅减少。
  • Claude 在你明确只需设计文档之前，就已经为退款功能编写了代码并引入了各种 import，白白浪费时间在无谓的构建错误上。
  • Claude 为服务类型新建了数据库字段，而非复用已有字段，迫使你中途纠正这一过度设计。
上下文核实不足
许多错误源于 Claude 在未阅读相关文档或配置文件之前，就对 API 结构、文件路径或框架行为做出了想当然的假设。明确请求 Claude 在提出修复方案前先读一读关键配置或 Schema 文件，可以有效缓解这一问题。
  • Claude 多次误判 Pinduoduo 和 JD API 的参数与响应结构，不得不参照官方文档与错误日志反复调试。
  • Claude 在 Hexo 路由与主题覆盖上频频受挫，尝试了多种错误的文件结构改动，最终才定位到根因——缺失一个重定向脚本。
调试循环
你常遇到这样的情况：Claude 的第一版修复要么不对，要么不完整，导致你陷入多轮调试，而若初始诊断能更彻底，这些问题本可一次性解决。鼓励 Claude 在给出修复方案前，先考虑边界情况与潜在副作用，有助于打破这类循环。
  • 时区修复首次失败，原因是 Claude 忽略了一个自定义的 LocalDateTimeTypeHandler 覆盖了 SQL 转换逻辑，不得不在第二轮迭代中将其移除。
  • 禁用过去时间段的逻辑，在你的反馈之后才从结束时间比较修正为开始时间比较，可见初始逻辑并未深思熟虑。

建议加入 CLAUDE.md 的规则

直接将以下内容粘贴到 Claude Code，即可加入你的 CLAUDE.md。

多个会话出现 CSS 改动不生效的情况，原因是独立的 production CSS 文件或缓存问题，导致额外的调试轮次。
多个摩擦点源于 Claude 新建了本可复用的 DB 字段，或在未查阅文档的情况下就误判了 API 参数。
多个会话涉及 Hexo 路由问题，脚本注入导致代码直接显示在首页而非被执行。
某个会话中，初次时区修复失败，正是因为自定义 TypeHandler 覆盖了转换逻辑，直到第二轮迭代才被发现。

直接将以下内容粘贴到 Claude Code，它将自动为你完成配置。

Custom Skills（自定义技能）
为 commit message、PR 评审、Bug 修复模板等重复性工作流，打造可复用的提示词。
对你的价值：你在 69 个会话中密集地进行调试与功能实现；像 /debug-backend 或 /fix-css 这样的 Skill，能让你的方法更规范，减少重复指令的输入。
Create .claude/skills/debug-backend/SKILL.md with steps: 1) Check logs, 2) Identify error type, 3) Propose fix, 4) Verify. Then run /debug-backend when hitting backend bugs.
MCP Servers
通过 Model Context Protocol 将 Claude 与外部工具、数据库、API 连接起来。
对你的价值：你经常调试 PDD/JD、DashScope、微信支付等 API 集成，并自行安装过 MCP Server；为常用 API 预先配置好 MCP 连接，可以大幅提升集成调试效率。
claude mcp add my-api-debug -- npx -y @modelcontextprotocol/server-fetch
Hooks（钩子）
在 post-edit、pre-commit 等特定生命周期节点，自动运行 Shell 命令。
对你的价值：你只有 3 次 commit，却有大量代码编辑；通过 Hook 在每次编辑后自动运行 TypeScript 检查或 lint，可以把编译错误（比如那个变量初始化的 Bug）提前拦截，而不必等到运行才发现问题。
Add to .claude/settings.json: {"hooks": {"post_edit": "npx tsc --noEmit"}}

Set up an MCP server for the Pinduoduo affiliate API so I can query endpoint specs and test responses directly from Claude.

前沿展望

AI 辅助开发正在从被动的代码生成，迈向能够端到端处理复杂工程任务的自主式多步工作流。

自主式测试驱动调试循环
Agent 可以自主为报告中的 Bug 编写失败测试，迭代修复直至所有测试通过，并验证无回归风险，全程无需人工介入。这将把调试从一项手动诊断工作，转变为一个自我修正的流水线，在代码进入评审之前即保证正确性。
入手建议：结合 Claude Code 与 Jest 或 JUnit 等测试框架，指示 Agent 先写测试，再修复代码直至全部通过。
粘贴到 Claude Code：
Analyze the bug report for [feature]. First, write a comprehensive test case that reproduces the issue and fails. Then, iteratively modify the source code until the test passes. Run the full test suite to ensure no regressions. Report the final diff and test results.
并行多服务集成 Agent
部署并行的 Sub-Agent，同时集成多个外部 API、协调数据 Schema、实时解决文档冲突。这将消除 API 集成的串行瓶颈，让复杂的多供应商功能在原先几分之一的时间内完成构建。
入手建议：借助 Claude Code 的 TaskCreate 工具为每个 API 端点生成并行任务，然后统一编排它们的输出。
粘贴到 Claude Code：
Split the integration of [API_A], [API_B], and [API_C] into three parallel tasks. Each task should: 1) Read official docs, 2) Implement the client wrapper, 3) Write unit tests against mocked responses. Once all three complete, merge them into a unified service layer with consistent error handling and data models.
自愈合 CI/CD 流水线助手
Agent 可以监控 CI 失败，通过分析日志与最近的提交定位根因，并针对依赖冲突、配置错误等常见问题自动提出或直接应用修复方案。这将打造一个韧性十足的开发循环，让基础设施层面的摩擦被自主化解。
入手建议：通过 Bash/Grep 工具将 Claude Code 接入你的 CI 日志，提示它分析失败原因并给出修复建议。
粘贴到 Claude Code：
Review the latest CI failure log at [path]. Identify the root cause by cross-referencing the error with recent git commits and dependency changes. If it's a known pattern (e.g., version mismatch, missing env var), generate the fix directly. If complex, provide a step-by-step remediation plan with exact commands.
